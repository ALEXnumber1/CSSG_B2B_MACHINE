# Skill: Marketing, Copywriting y Neuromarketing para Seguridad B2B

## 1. Fórmulas de Copywriting

### PAS (Problem → Agitation → Solution)
1. **Problem**: Identifica el dolor del lector ("Su empresa está expuesta")
2. **Agitation**: Amplifica la consecuencia ("Un solo incidente puede costar millones")
3. **Solution**: Presenta CSSG como la solución ("Nuestro sistema reduce incidentes en un 60%")

### AIDA (Attention → Interest → Desire → Action)
1. **Attention**: Titular impactante con dato concreto
2. **Interest**: Contexto que conecta con su realidad
3. **Desire**: Prueba social + beneficios tangibles
4. **Action**: CTA claro y de baja barrera ("Evalúe gratis", no "Compre ahora")

### 4U (Urgent, Unique, Ultra-specific, Useful)
Para títulos de blog:
- **Urgente**: "Antes de que ocurra un incidente..."
- **Único**: "La metodología que usan las embajadas del G7..."
- **Ultra-específico**: "5 errores que cometen el 80% de las empresas..."
- **Útil**: "Guía completa para reducir un 40% sus costos..."

## 2. Neuromarketing B2B — Triggers de Decisión

### Los 6 principios de Cialdini aplicados a seguridad:

| Principio | Aplicación CSSG |
|---|---|
| **Autoridad** | ISO 9001, +17 años, clientes diplomáticos del G7 |
| **Prueba Social** | "80% de éxito en licitaciones", "+12 años sin incidentes" |
| **Escasez** | "Solo atendemos X proyectos simultáneamente" |
| **Reciprocidad** | Herramienta de riesgo GRATIS, informe PESTEL gratis |
| **Compromiso** | Micro-conversiones: descarga → análisis → consulta → contrato |
| **Simpatía** | Humanizar: oficiales destacados, equipo con nombre, historias |

### Bias cognitivos útiles:
- **Aversión a la pérdida**: "Lo que pierde sin seguridad" > "Lo que gana con seguridad"
- **Efecto de anclaje**: Mostrar precio alto primero, luego precio competitivo de CSSG
- **Sesgo de autoridad**: Mencionar ISO, G7, normas internacionales
- **Efecto halo**: Si protegemos embajadas del G7, somos buenos para cualquiera

## 3. Storytelling B2B — Estructura Narrativa

### Framework del Héroe Transformado:
1. **Situación inicial**: Empresa con seguridad básica o deficiente
2. **Detonante**: Incidente, auditoría fallida, licitación perdida
3. **Guía**: CSSG aparece como consultor/aliado
4. **Transformación**: Implementación del plan de seguridad
5. **Resultado**: Datos medibles (reducción de incidentes, ahorro, licitación ganada)

### Reglas de storytelling en seguridad:
- NUNCA inventar casos de éxito (usar datos reales anonimizados)
- Usar la tercera persona para casos sensibles
- Conectar emocionalmente: "La tranquilidad de saber que..." 
- Cerrar con data dura, no con sentimientos

## 5. SEO — Estructura de Artículo Óptima

```
H1: Keyword principal + beneficio (máx 60 caracteres)
Meta description: 155 caracteres con keyword + CTA implícito

Intro: Hook con dato o pregunta (PAS: Problem)
  - 2-3 párrafos cortos, keyword en el primer párrafo

H2: Subtema 1 (keyword secundaria)
  - Contenido útil, listas, datos
  
H2: Subtema 2
  - Casos de ejemplo (storytelling)

H2: Subtema 3
  - Comparativas, tables, datos duros

H2: CTA (lo que CSSG ofrece para resolver esto)
  - Link a herramienta de riesgos o consultoría

FAQ (Schema markup opcional):
  - 3-5 preguntas que la gente busca en Google
```
